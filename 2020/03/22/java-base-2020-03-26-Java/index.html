Java异常

Throwable是java语言中所有错误与异常的超类,下一层是Error和Exception

  1. Error:是指在Java运行过程中资源耗尽或者系统内部错误,系统不会跑出该类额异常,如果出现这种错误,除了告知用户剩下的就是尽力终止程序.不可控
  2. Exception:分为两个体系1.运行时异常 (RuntimeException)2.受检异常(CheckedException)
  1. 运行时异常:ArithmeticException,ArrayIndexOutOfBoundsException,ClassCaseException,NullPointerException,IllegalArgumentException
  2. 受检异常:ClassNotFountException,NuSuchFiledException,NuSuchMethodException,InterruptedException
Throw和throws的区别
  1. throws是声明异常,让调用者知道该功能可能出现问题.throw是跑出具体的异常,执行到throw,功能就结束了.
  2. throws表示出现异常的可能,throw一定跑出异常.
  3. 两者都是消极的处理方式.还是抛出可能出现的异常,但是不会有函数处理
Java反射机制的概念

java反射机制是指在运行状态中,对于意一个类都知道这个类所有的方法和属性.并且对于任意一个对象都能调用它的任意方法.

Java反射API
  1. Class类,反射的核心,可以获取类的属性方法等信息
  2. Field类,Java.lang.reflect包中到了类,表示类的成员变量,可以设置或者获取类中属性的值
  3. Method类:Java.lang.refelct包中的类,表示类的方法,它可以用来设置获取类中的方法信息.
  4. Constructor类,Javas.lang.refletc包中的类,表示类的构造方法.
